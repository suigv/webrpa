# AI 工作流设计清单 (Checklist)

本清单旨在指导开发者如何设计基于 LLM（大语言模型）和 VLM（视觉语言模型）的稳健 RPA 工作流。

---

## 1. 感知层：UI 观察策略

### 1.1 结构化状态优先 (Structured State First)
- [ ] 是否优先尝试使用 `ui.match_state` 获取结构化 UI 树？
- [ ] 是否为关键 UI 元素定义了稳定的 `state_id`？
- [ ] 观察超时时间是否合理（建议针对动态加载页面设置 5-10s）？

### 1.2 视觉回退机制 (Vision Fallback)
- [ ] 当结构化观察失败时，是否配置了 `fallback_modalities`（如 `vlm` 或 `vision`）？
- [ ] 是否确保在视觉回退时，系统能够捕获当前屏幕截图？
- [ ] AI 是否具备足够的上下文来理解图像（例如：告知 AI 屏幕的分辨率）？

---

## 2. 推理层：任务规划与决策

### 2.1 目标定义 (Goal Definition)
- [ ] 任务目标是否具体且可衡量？（避免“浏览网页”，改用“找到并点击登录按钮”）
- [ ] 是否告知 AI 任务成功的终止条件？

### 2.2 动作空间限制 (Action Space Control)
- [ ] 是否通过 `allowed_actions` 严格限制了 AI 的权限？
- [ ] 是否排除了高风险动作（如 `device.reboot`），除非业务确实需要？

### 2.3 历史上下文 (History Context)
- [ ] 是否为 AI 提供了前几步的操作历史？
- [ ] 历史记录是否包含了失败的尝试，以防止 AI 陷入重复操作的死循环？

---

## 3. 执行层：动作执行与反馈

### 3.1 坐标与缩放
- [ ] AI 输出的点击坐标是否经过了归一化处理？
- [ ] 是否考虑了不同设备屏幕比例导致的点击偏移？

### 3.2 动作验证 (Verification)
- [ ] 执行动作后，是否再次观察 UI 状态以确认效果？
- [ ] 是否引入了“裁判（Judge）”模式，通过独立请求判断上一步是否成功？

---

## 4. 容错与熔断 (Safety & Circuit Breaker)

### 4.1 步数限制 (Step Budget)
- [ ] 是否设置了 `max_steps` 以防止 AI 在无法处理的页面无限循环？

### 4.2 状态停滞检测 (Stagnancy Detection)
- [ ] 是否开启了 `stagnant_limit`？（当 UI 状态连续 N 次无变化时强制中止）

### 4.3 重试与退避 (Retry & Backoff)
- [ ] 系统是否能识别 AI 返回的“重试”建议？
- [ ] 面对网络超时或模型 API 错误，是否有指数退避机制？

---

## 5. 调试与审计 (Traceability)

### 5.1 决策轨迹记录
- [ ] 所有 AI 的 Thought、Action 和 Observation 是否已记录在 `ModelTraceStore` 中？
- [ ] 轨迹是否包含截图引用，以便人工回溯“AI 为什么这么想”？

### 5.2 离线蒸馏
- [ ] 成功的 AI 轨迹是否可以导出并蒸馏为确定性的 YAML 插件？
