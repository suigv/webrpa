# Frontend

当前前端是 `web/` 下的独立 Vite 工程。后端不是前端静态文件服务器。

控制台当前采用单页分栏结构：

- 顶部为品牌区、系统状态区和主导航。
- 主导航当前按业务语义组织为：`指挥台`、`任务设计`、`执行队列`、`资源仓库`、`系统偏好`。
- 主内容区保留统一总览 Hero，但默认操作入口当前收敛到 `指挥台`，而不是先让用户进入一组并列功能页。
- `指挥台` 当前按三条业务泳道组织：
  - `Fleet`：节点态势与批量选择
  - `Mission`：任务设计、执行与资源补齐的下一步入口
  - `Intervention`：运行形态、系统级输出与人工接管提示
- 设备视图会在页面总览区和 `指挥台` 同步展示节点总量、在线数、占用数和已锁定数。
- `任务设计` 当前整合了统一 AI 入口说明、任务设计、历史复用、执行协作、草稿列表/详情，以及 AI 相关运行洞察与蒸馏进度。
- `任务设计` 当前保留双模式结构：默认引导模式负责按步骤完成任务图设计，高级模式负责查看完整上下文并快速调整。

## 本地开发

后端：

```bash
./.venv/bin/python -m uvicorn api.server:app --host 127.0.0.1 --port 8001
```

前端：

```bash
cd web
npm install
npm run dev
```

如需无 RPC 的纯 Web 路径：

```bash
MYT_ENABLE_RPC=0 ./.venv/bin/python -m uvicorn api.server:app --host 127.0.0.1 --port 8001
```

## 部署边界

- 前端构建产物位于 `web/dist/`。
- 推荐由 Nginx 或其他静态服务器托管 `web/dist/`。
- 后端保留 `/web` 作为入口路由：
  - 配置 `MYT_FRONTEND_URL` 时重定向到前端地址。
  - 未配置时返回部署提示。
- 仓库当前不再提供 `webplayer` 静态播放页；人工接管仍走任务接管接口和设备轻控制接口。

## AI 对话当前界面

- AI 相关的主导航入口当前是 `AI 工作台`，并且它是唯一的任务图设计主入口。
- AI 工作台当前保留两种模式：
  - `引导模式`：默认进入，按步骤完成“目标与资源 -> 任务描述与约束 -> 确认任务图 -> 执行与回流”
  - `高级模式`：查看完整上下文并快速调整，保留输入区、任务图主画布和参考上下文区
- 双模式当前必须共用同一套设计输入、任务图草案、历史会话引用、确认状态和执行状态；模式切换不应丢失当前任务上下文。
- AI 工作台任务图设计输入区当前应为每个关键输入项提供常驻示例文本，不仅依赖 `placeholder`，以便用户在不清楚填写方式时可直接参考。
- AI 工作台在“绑定账号”之后当前应展示“挑战处理能力”区块：
  - `自动 2FA 验证码` 为已可用能力；当任务匹配支持 TOTP 的流程且所选账号包含 2FA 密钥时，应明确显示会自动生成 6 位验证码。
  - `自动 2FA 验证码` 当前默认开启，但用户可在工作台里显式关闭；关闭后本次执行不会把账号资源中的 2FA 密钥带入运行链路。
  - `人工接管` 为已可用能力，用于图形验证码、滑块验证码、短信码、邮箱码或风险确认等尚未自动化的挑战。
  - 其他高级挑战能力当前只作为前端占位和文档说明展示，但状态命名必须与真实实现方向一致：
    - 图形验证码、滑块验证码可标注为 `AI 驱动`
    - 短信码、邮箱码应标注为 `通道驱动` 或同类表达，不得误写成 `AI 驱动`
  - 上述占位都不得伪装成当前已自动可调用能力。
- AI 工作台和设备详情 AI 对话当前都应提供明确的账号需求声明：
  - 默认视为“需要账号数据”
  - 用户可显式勾选“该任务无需账号数据”
  - 仅在任务不依赖账号、密码、2FA、token 等资源时才允许跳过账号绑定
- 任务图草案当前仍为只读展示，用于确认控制流、声明脚本、成功/失败出口、人工接管点和执行门槛；当前不支持节点级手工编辑。
- AI 工作台任务图摘要与设备详情 AI 对话的执行摘要当前都应显式展示本次挑战处理策略，至少包含：
  - 是否会自动使用账号 2FA 密钥生成验证码
  - 当前仍需人工接管的挑战类型
- 历史 AI 会话当前只保留两种主要语义：作为当前设计参考、继续编辑草稿。
- AI 工作台当前仍是 `agent_executor` 的统一下发入口，并在成功后进入统一执行视图。
- 设备详情中的 `AI 对话` 接口继续保留，但当前只负责单设备快速发起、执行观察和人工接管；任务图设计、历史复用和蒸馏沉淀统一收口到 AI 工作台。
- 设备详情中的 `AI 对话` 弹窗当前也应提供与 AI 工作台一致的 `自动使用账号 2FA 密钥生成验证码` 开关：
  - 默认开启
  - 关闭后本次单机执行不会把账号资源中的 2FA 密钥带入运行链路
  - 提示文案应根据登录目标、账号绑定与 2FA 字段是否存在动态变化
- 设备视图中的 AI 对话弹窗当前不会在输入期自动规划；执行摘要会在真正准备执行时生成，用于当前单任务执行判断。
- AI 对话弹窗和设备详情轻控制当前会在 SSE 事件、接管请求和人工轻控制 trace 中持续透传 `currentDeclarativeStage`，保证人工接管时仍知道当前停留在哪个声明阶段。

## 前端任务提交契约

- 任务表单应来自 `GET /api/tasks/catalog`。
- 插件提交必须遵守 `manifest.inputs` 白名单。
- `targets` 承载目标设备上下文。
- 不要让前端重新把 `device_ip`、`package`、目标元数据之类的运行时字段塞回插件 `payload`。

## 鉴权

启用 JWT 时：

- HTTP 使用 `Authorization: Bearer <jwt>`
- 浏览器 WebSocket 使用 `Sec-WebSocket-Protocol: bearer.<jwt>`
