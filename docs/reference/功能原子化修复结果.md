# 功能原子化修复结果

更新时间：2026-03-07

## 本轮结论

当前分支已经完成一轮面向原子化边界的实际修复，重点解决了动作层热点模块膨胀、selector 资源清理、共享 JSON 写入安全、以及 facade/internal split 这几类问题。

这不是“重写架构”，而是在保留现有插件化运行时主干的前提下，对高风险热点做收敛。

## 已完成事项

### 1. Selector / RPC 清理补全

- `engine/interpreter.py`
- `engine/actions/ui_actions.py`
- `engine/actions/_ui_selector_internals.py`

完成内容：

- `ui.create_selector` 保持 selector 持有的 RPC 会话可继续使用
- `Interpreter.execute()` 在 `finally` 路径自动调用 selector 清理
- workflow 即使未显式执行 `ui.selector_free`，也不会长期泄露 selector-backed RPC 资源

### 2. JSON 原子写入与共享状态加固

- `core/data_store.py`
- `engine/actions/sdk_actions.py`
- `engine/actions/_sdk_shared_store.py`

完成内容：

- 引入 `write_json_atomic`，避免直接写文件造成损坏 JSON
- `save_shared()` 改为走共享 store 更新路径，而不是裸的读改写
- 增加同进程并发保护，避免线程级 read-modify-write 丢写

### 3. 跨进程共享写入加固

- `engine/actions/_sdk_shared_store.py`
- `tests/test_sdk_actions_runtime.py`

完成内容：

- 为共享 store 增加 sibling lockfile + `fcntl.flock(LOCK_EX)`
- 锁覆盖整个 read -> modify -> write 临界区
- 锁文件改为持久化，不再在 unlock 后删除，避免 inode 重建竞争
- 新增 multiprocessing 回归测试，覆盖跨进程竞争写入场景

### 4. `ui_actions` facade 拆分

- `engine/actions/ui_actions.py`
- `engine/actions/_ui_selector_internals.py`

完成内容：

- `ui_actions.py` 继续保留稳定 facade 和 monkeypatch surface
- selector / node 细节下沉到 `_ui_selector_internals.py`
- 测试与 action registry 依赖的外部导入面保持兼容

### 5. `sdk_actions` facade 拆分

- `engine/actions/sdk_actions.py`
- `engine/actions/_sdk_bindings_internals.py`
- `engine/actions/_sdk_shared_store.py`

完成内容：

- `sdk_actions.py` 保持 registry/test 使用的稳定入口
- SDK method binding、参数构造、shared-store 逻辑拆到内部模块
- `sdk.*` / `mytos.*` action 名称保持不变

## 分支上判定为“不适用”的事项

以下原计划项在当时的 remediation worktree 中被判定为“不适用”或已由后续工作单独处理：

- Task 1：共享 RPC bootstrap helper（后续已完成）
- Task 4：task failure feedback hook 抽离（后续已完成）
- Task 7：login fallback composite actions
- Task 8：`x_mobile_login` workflow 压缩

原因不是放弃，而是当前分支代码形态与原计划来源分支不一致：

- 本分支没有 `engine/actions/state_actions.py`
- 当前 `plugins/x_mobile_login/script.yaml` 是简化状态契约版本，不存在原计划里那种超长 fallback workflow
