# 功能原子化修复结果

更新时间：2026-03-08

## 本轮结论

当前分支已经完成一轮面向原子化边界的实际修复，重点解决了动作层热点模块膨胀、selector 资源清理、共享 JSON 写入安全、以及 facade/internal split 这几类问题。

这不是“重写架构”，而是在保留现有插件化运行时主干的前提下，对高风险热点做收敛。

## 已完成事项

### 1. Selector / RPC 清理补全

- `engine/interpreter.py`
- `engine/actions/ui_actions.py`
- `engine/actions/_ui_selector_internals.py`

完成内容：

- `ui.create_selector` 保持 selector 持有的 RPC 会话可继续使用
- `Interpreter.execute()` 在 `finally` 路径自动调用 selector 清理
- workflow 即使未显式执行 `ui.selector_free`，也不会长期泄露 selector-backed RPC 资源

### 2. JSON 原子写入与共享状态加固

- `core/data_store.py`
- `engine/actions/sdk_actions.py`
- `engine/actions/sdk_shared_store_support.py`

完成内容：

- 引入 `write_json_atomic`，避免直接写文件造成损坏 JSON
- `save_shared()` 改为走共享 store 更新路径，而不是裸的读改写
- 增加同进程并发保护，避免线程级 read-modify-write 丢写

### 3. 跨进程共享写入加固

- `engine/actions/sdk_shared_store_support.py`
- `tests/test_sdk_actions_runtime.py`

完成内容：

- 为共享 store 增加 sibling lockfile + `fcntl.flock(LOCK_EX)`
- 锁覆盖整个 read -> modify -> write 临界区
- 锁文件改为持久化，不再在 unlock 后删除，避免 inode 重建竞争
- 新增 multiprocessing 回归测试，覆盖跨进程竞争写入场景

### 4. `ui_actions` facade 拆分

- `engine/actions/ui_actions.py`
- `engine/actions/_ui_selector_internals.py`

完成内容：

- `ui_actions.py` 继续保留稳定 facade 和 monkeypatch surface
- selector / node 细节下沉到 `_ui_selector_internals.py`
- 测试与 action registry 依赖的外部导入面保持兼容

### 5. `sdk_actions` facade 拆分

- `engine/actions/sdk_actions.py`
- `engine/actions/sdk_runtime_support.py`
- `engine/actions/sdk_config_support.py`
- `engine/actions/sdk_shared_store_support.py`
- `engine/actions/sdk_profile_support.py`
- `engine/actions/sdk_business_support.py`

完成内容：

- `sdk_actions.py` 保持 registry/test 使用的稳定入口
- SDK method binding 保持在 facade，shared-store、运行时辅助、配置读取、profile 与轻业务逻辑拆到 `sdk_*_support.py` helper 模块
- `sdk.*` / `mytos.*` action 名称保持不变

### 6. `x_mobile_login` 定向 fallback-chain 压缩

- `plugins/x_mobile_login/script.yaml`

完成内容：

- 复用 `ui.click_selector_or_tap` 收敛登录入口行与登录提交路径的 selector 点击回退
- 复用 `ui.focus_and_input_with_shell_fallback` 收敛账号、密码与 2FA 输入路径的重复输入 fallback
- 保持原有状态判断、stop message、分支标签与插件整体结构稳定，不做整插件重写

### 7. `/api/runtime/execute` debug-only 语义澄清

- `api/server.py`
- `tests/test_runtime_execute_integration.py`

完成内容：

- OpenAPI 描述明确 `/api/runtime/execute` 是 debug/internal-only 同步直跑入口
- 该入口返回原始 Runner 结果，不创建 `/api/tasks` 托管任务记录
- 回归测试固定它不会参与 `/api/tasks` 生命周期字段、任务记录或 metrics artifacts

## 分支上判定为“不适用”的事项

以下原计划项在当时的 remediation worktree 中被判定为“不适用”或已由后续工作单独处理；其中部分事项现已在后续任务中落地：

- Task 1：共享 RPC bootstrap helper（后续已完成）
- Task 4：task failure feedback hook 抽离（后续已完成）

原因不是放弃，而是当前分支代码形态与原计划来源分支不一致：

- 早期 remediation worktree 的判断口径与当前主分支代码形态并不是同一时间点
- 当时的 `plugins/x_mobile_login/script.yaml` 判断口径与后续落地的最小安全压缩并不是同一时间点的状态

目前这两项的最新结论是：

- login fallback composite actions 已在 `x_mobile_login` 的主路径上做定向复用
- `x_mobile_login` workflow 压缩已完成最小安全收口，但仍保留为 watchpoint，而不是宣称整插件重写
